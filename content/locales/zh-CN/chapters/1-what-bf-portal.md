# Battlefield Portal 是什么？

> 官方准备的“面向玩家的开发套件”

提到 Battlefield 系列，很多人首先想到的，应该是贯穿整个系列的“战争体验的真实感”，例如宏大的战场、小队协作、可破坏的环境等。
不过，在这个系列中，2021 年登场的 Battlefield Portal（以下简称 Portal）是 Battlefield 历史上最具实验性，也最具革新性的机制之一。

Portal 是由 DICE 和 Ripple Effect Studios 开发的一组官方工具，目的是让玩家亲手为 Battlefield 创建规则。
换句话说，Portal 不只是用来“游玩”的战场，也是用来“设计”的战场。

## 官方定义（来自 EA 官方网站）

> Battlefield Portal 是一个巨大的沙盒，创作者和玩家可以在其中将 Battlefield 推向极限。
> 移动、缩放、复制对象，以前所未有的自由度控制环境。
> 使用 NPC 脚本和可自定义 UI，创建完全独特的游戏模式。
> 传奇的 Battlefield 体验正在等待被创造。展现你的真正价值吧。
>
> 思考、创造、游玩：
> 这种独特的体验被称为“社区体验”。你可以重现类似 Horde Mode 的经典游戏模式，也可以创造俯视角 2D 射击游戏这样的创新体验。
>
> 强大的创作者工具：
> Portal 搭载了空间编辑（地图/地理编辑）功能，以及用于自定义 AI 战斗人员行为的 AI 脚本功能。Portal Builder 还支持自定义 UI 脚本，用来构建完全可自定义的用户界面。
>
> 用你自己的方式体验 Battlefield：
> 借助网页创作工具中的 Portal 工具、丰富的修改器和规则组选项，你可以用自己喜欢的方式体验 Battlefield。硬核模式、限定武器装备等，可能性是无限的。
>
> （来源：EA 官方《PC 的力量与 PORTAL 的可能性相融合》）

## 这个定义意味着什么

这段官方说明表明，Portal 并不只是一个“规则设置工具”，而是**一个拥有接近游戏开发环境自由度的创作工具**。其中尤其值得注意的是以下三个要素。

### 对象的空间操作
* 现有 `Static` layer 中包含的地形和已烘焙资源无法编辑。
* 创作者可以在 Spatial Editor 中追加放置当前 Level 可用的资源，并通过移动、旋转、缩放已放置的对象来构建空间。
  → 也就是说，Portal 的地图编辑不是“改写现有地形”，而是“添加可用部件，组装出一种体验”。

### AI 脚本和自定义 UI
* 可以自行制作 NPC（AI 士兵）的行为模式和 UI 结构。
  → 可以设计游戏体验本身。

### 修改器和规则组
* 可以通过 GUI 轻松设置无限弹药、生命值倍率、限定武器等规则变量。
  → 从“修改”现有模式到“重新构建”现有模式都能应对。

这样一来，Portal 将 Battlefield 的核心“战场模拟”释放出来，变成了扩展玩家想象力的创作场。

## 与过去系列作品的区别

过去的 Battlefield 作品（BF3、BF4、BFV 等）中，也可以通过“自定义服务器”或“服务器规则”的形式进行一定程度的修改。
不过，这些修改主要停留在地图顺序、武器限制、重生设置等范围内。

而在 Portal 中，你可以从结构层面重写这些概念。
换句话说，**玩家可以站在接近开发者的视角，操作游戏模式的内部逻辑**。

这正是 Portal 最大的不同，也是它最具革命性的地方。

## “社区体验”这个新的单位

EA 在说明 Portal 时反复使用“社区体验（Community Experience）”这个词。
它指的是**由普通玩家制作的地图、规则、AI、UI 组合而成的一整个游戏体验**。

过去只是“更改服务器设置”的东西，在 Portal 中会被当作一个作品单位来处理。
其他玩家可以搜索这个“作品”、游玩它、评价它，并留下评论。
这就像是**游戏版 YouTube：玩家上传并分享自己制作的“体验”**。

## Portal 是官方认可的“创作场”

传统的 Battlefield 是“接收体验”的游戏，而 Portal 是“创造体验”的游戏。

AI 脚本、自定义 UI、空间编辑、规则构建。当这些元素融合在一起时，玩家就不再只是士兵，而是设计自己世界的创作者。

如果要为 Portal 下一个定义，可以说它是“官方提供的、打破开发者与玩家边界的实验场”。放在当下，这应该是最准确的说法。

# “Portal Custom”和“现有模式”的区别

在 Portal 中创建新体验时，最开始要选择的是 **Portal Custom** 或 **现有模式**。
这个选择会影响之后能使用的编辑器、自由度、发布时自动显示的信息，以及经验值（XP）的处理方式。下面具体说明各自的特点。

## A. Portal Custom：重新制作规则和地图

### 可以做什么

* 规则编辑器：可以组装条件分支、计数、触发器等游戏行为。
* 地图编辑器：可以通过放置、移动、缩放、复制对象等方式编辑空间本身。
* UI / AI：可以用脚本调整画面显示（UI）和 AI 行为，大幅改变游玩手感。

### 发布时的显示方式

* 体验详情中可能会显示“自定义逻辑”等标记。不过，创作者并不能自由添加标签。
* 搜索和加入的路径，会受到分享代码、标题、256 字以内的说明文、缩略图、外部宣传方式的影响。

### 经验值（XP）

* 原则上，请先假设会受到 XP 限制。
* 尤其是自定义逻辑、独自设计的 AI、极端提高得分效率或击杀效率的设置，往往容易被当作与普通进度奖励不同的内容处理。
* 不要以 XP 为目的，而要以体验或活动为目的来设计，并在发布前确认最新的官方说明。

### 适合的用途

* 不受现有框架限制的新作品、合作（对 AI）玩法、竞速、小游戏等实验，以及活动中的“看点”设计。

## B. 现有模式：在标准功能范围内调整

#### 可以做什么

* **不能使用规则编辑器 / 地图编辑器。**
* 取而代之的是，可以使用标准自定义项目进行广泛调整，例如 AI 出现、武器和装备限制、重生、票数、目标数变更等。
* 体感上更接近“继承 BF2042 时代的 Portal，但不进行规则编辑”。

### 发布时的显示方式

* 因为手感接近标准模式，初次参加的玩家更容易进入。
* 如果在标题、说明文、缩略图中明确写出意图，例如硬核、AI 训练、面向新手等，初次看到的人会更容易判断内容。

### 经验值（XP）

* 根据设置内容和官方运营方式，也可能受到限制。
* 如果条件极端，例如短时间内很容易大量击杀，就更容易成为限制对象。

### 适合的用途

* 以现有规则为基础、易于加入的常设服务器，AI 练习，武器限制，硬核设置等。

## 比较要点

| 角度 | Portal Custom | 现有模式 |
| ---- | ---- | ---- |
| 编辑器 | 有规则 / 地图编辑器 | 无（用标准项目调整） |
| 自由度 | 高（重新设计行为和空间） | 中（标准范围内） |
| AI / 武器限制 | 可以（也能调整行为） | 可以（用标准功能设置） |
| UI / 演出 | 可以追加自定义 UI 和演出 | 以默认显示为主 |
| XP | 可能受到限制 | 可能受到限制 |
| 主要用途 | 新作品、新体验游戏、活动 | 常设运营、面向入门、稳定运营 |

## 实务技巧：选择时的判断标准

* 重视参与门槛：先用现有模式固定核心玩法。
* 重视独特性：看清需求之后，再用 Portal Custom 完成最终设计。
* 说明文要以 256 字以内为前提，简短写清“目的、推荐人数、所需时间”，方便初次看到的人判断。
* **定期举办（星期几、几点）和社区宣传（X / Discord 等）一起做，会更容易固定下来。**
