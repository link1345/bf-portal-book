# 游戏概述

`AcePursuit` 是使用飞机进行竞赛/计时生存的示例。

它拥有大型游戏模式所需的所有元素，例如检查点、圈数、车辆选择、排名 UI、开始/结束倒计时等。

# 主要内容

* 在 `RaceTrack`、`Checkpoint`、`Vector3` 定义课程数据。
* 在 `GameType` 上切换竞赛和时间生存。
* 在 `TrackData` 管理整体比赛状态、参与者、获胜者和倒计时。
* 在 `PlayerProfile` 上管理每个玩家的车辆、圈数、时间和 UI 状态。
* 在UI组中显示车辆选择、排名、圈数和剩余时间。
* 使用 SFX 和 WorldIcon 显示检查点通道和状态变化。
* 检查叛逃和残疾玩家并清理 UI 和车辆。

# 阅读顺序

## 1.查看课程定义

首先看一下`RaceTrack`、`Checkpoint`、`tracks`的流程。

在飞机比赛中，首先重要的是去哪里、按什么顺序以及跑多少圈。请确保您有检查点位置、方向和校正标志作为数据。

## 2.查看轨迹数据

`TrackData` 是具有一个种族的状态价值的核心类别。

参赛者、获胜者、起始数、结束数、游戏类型、可选择的飞机等都聚集在这里。如果您想知道比赛现在在哪里，请从关注 `TrackData` 开始。

## 3. 查看玩家档案和 UI

每个玩家的状态都发布在 `PlayerProfile` 上。

每个玩家的车辆选择、驾驶时间、圈数、排名和记分牌显示都会发生变化。如果你阅读 `TrackData` 来了解整体，`PlayerProfile` 来了解个人，以及 UI 类来了解演示，那么你就很难迷失方向。

## 4.查看终止处理和退出处理

`Winner`、`PlayerCompletedTrack`、`PlayerLeftGame`等是决定比赛如何结束的重要流程。

对于较大的模式，终止过程比启动过程更脆弱。请检查确定获胜者的顺序，所有人完成比赛，中途离开比赛，关闭UI，丢弃车辆。

# 提示

`AcePursuit` 在开头附近分组了常量。

通过将 `VERSION`、`debugPlayer`、`MinimumPlayerToStart`、`MapPlayers` 等调整值以及各种倒计时值放在顶部，将更容易区分测试版本并调整平衡。

值得注意的是，种族和时间生存是由 `GameType` 分隔的。当游戏类型数量增加时，与其到处写条件分支，不如修复表示状态和类型的值，这样会更容易阅读。

# 结论

`AcePursuit` 是读取带有状态的较大游戏模式的示例。

不要从一开始就深入了解 UI 的细节，只需按照课程定义、`TrackData`、`PlayerProfile`、获胜者确认和退出处理即可。它还可以应用于比赛以外的模式，例如按顺序通过目标的模式，或者每个参与者都有进步的模式。
