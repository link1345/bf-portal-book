# 遊戲概述

`AcePursuit` 是使用飛機進行競賽/計時生存的範例。

它擁有大型遊戲模式所需的所有元素，例如檢查點、圈數、車輛選擇、排名 UI、開始/結束倒數計時等。

# 主要內容

* 在 `RaceTrack`、`Checkpoint`、`Vector3` 定義課程資料。
* 在 `GameType` 上切換競賽和時間生存。
* 在 `TrackData` 管理整體比賽狀態、參與者、獲勝者和倒數計時。
* 在 `PlayerProfile` 上管理每位玩家的車輛、圈數、時間和 UI 狀態。
* 在UI組中顯示車輛選擇、排名、圈數和剩餘時間。
* 使用 SFX 和 WorldIcon 顯示檢查點通道和狀態變化。
* 檢查叛逃和殘疾玩家並清理 UI 和車輛。

# 閱讀順序

## 1. 查看課程定義

首先先來看看`RaceTrack`、`Checkpoint`、`tracks`的流程。

在飛機比賽中，首先重要的是要去哪裡、按什麼順序、跑多少圈。請確保您有檢查點位置、方向和校正標誌作為資料。

## 2. 查看軌跡數據

`TrackData` 是具有一個種族的狀態價值的核心類別。

參賽者、獲勝者、起始數、結束數、遊戲類型、可選擇的飛機等都聚集在這裡。如果您想知道比賽現在在哪裡，請從關注 `TrackData` 開始。

## 3 . 檢視玩家檔案和 UI

每位玩家的狀態都發佈在 `PlayerProfile` 上。

每個玩家的車輛選擇、駕駛時間、圈數、排名和記分板顯示都會改變。如果你閱讀 `TrackData` 來了解整體，`PlayerProfile` 來了解個人，以及 UI 類別來了解演示，那麼你就很難迷失方向。

## 4. 查看終止處理和退出處理

`Winner`、`PlayerCompletedTrack`、`PlayerLeftGame`等是決定比賽如何結束的重要流程。

對於較大的模式，終止過程比啟動過程更脆弱。請檢查確定獲勝者的順序，所有人完成比賽，中途離開比賽，關閉UI，丟棄車輛。

# 提示

`AcePursuit` 在開頭附近分組了常數。

將 `VERSION`、`debugPlayer`、`MinimumPlayerToStart`、`MapPlayers` 等調整值以及各種倒數值放在頂部，將更容易區分測試版本並調整平衡。

值得注意的是，種族和時間生存是由 `GameType` 分隔的。當遊戲類型數量增加時，與其到處寫入條件分支，不如修復表示狀態和類型的值，這樣會更容易閱讀。

# 結論

`AcePursuit` 是讀取具有狀態的較大遊戲模式的範例。

不要從一開始就深入了解 UI 的細節，只需按照課程定義、`TrackData`、`PlayerProfile`、獲勝者確認和退出處理即可。它也可以應用於比賽以外的模式，例如按順序通過目標的模式，或每個參與者都有進步的模式。
