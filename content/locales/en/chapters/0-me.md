## Purpose of this book

The purpose of this book is for people who want to try doing something interesting with BattleField 6's Portal to learn how to use the basic editor as well as Portal's unique mechanisms.
The aim is to master the Portal's scripting function and map editor, including improving programming to operate the Portal server for a long time.
Please note that we do not deal with visual programming using blocks, but instead focus on implementation in TypeScript.

### About visual programming and map editors

Regarding visual programming and map editors, ["BF Portal BF6 Memorandum by.TOKAI_Server_JPN"](https://w.atwiki.jp/tokaiserver/pages/1.html) has a rich description.

## Target audience

To put it simply, the target audience is "beginner programming" level. Specifically, I am assuming the following.

* Have studied programming (at a level equivalent to studying at a Japanese junior high school in 2025)
* I've done programming in other languages, but I haven't done much in depth with other languages.

The definition of junior high school learning in Japan in 2025 is to be able to understand the mechanisms of measurement and control systems and to be able to create safe and appropriate programs.

> Guidance on computerization of education - Supplementary edition - (June 2020) Chapter 3 (Ministry of Education, Culture, Sports, Science and Technology): https://www.mext.go.jp/content/20200608-mxt_jogai01-000003284_004.pdf

Therefore, **this book is not an introductory book for programming or learning grammar**.
For TypeScript grammar, we recommend an introductory book such as "[Survival TypeScript](https://typescriptbook.jp/)".
We also recommend that you learn how to use Godot by reading "[GodotEngine Official Docs](https://docs.godotengine.org/ja/4.x/)".

### Target audience for the appendix

* People who can program as if they were breathing.

The appendix is a **handbook for people who can program but are new to Portal's TypeScript API**.
